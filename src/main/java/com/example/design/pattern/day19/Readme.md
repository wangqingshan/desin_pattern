**迭代器模式**

- 概述

可以存储多个成员对象，这些类通常称为聚合类（Aggregate Class），对应的对象称为聚合对象。为了更加方便的操作这些聚合对象
，同时可以灵活的位聚合对象增加不同的遍历方法，可以访问一个聚合对象中的元素但又不需要暴露他的内部结构。这就是迭代器模式。
通过引入迭代器，客户端无须了解聚合对象的内部结构即可实现对聚合对象中成员的遍历。还可以根据需要很方便的增加新的遍历方式。

聚合对象有两个职责：一是存储数据；二是遍历数据。从依赖性来看，前者是聚合对象的基本职责；而后者既可以是变化的，又是可分离的。
因此，可以将遍历数据的行为从聚合对象中分离出来，封装在一个被称为“迭代器”对象中，由迭代器提供遍历聚合对象的内部数据行为。
这将简化聚合对象的设计，更符合“单一职责原则”的要求。

- 定义

迭代器模式（Iterator Pattern）:提供一种方法来访问聚合对象，而不用暴露这个对象的内部表示，其别名为游标（Cursor）

- 迭代器模式的几个角色

1.Iterator(抽象迭代器)：它定义了访问和遍历元素的接口，声明了用于遍历数据元素的方法，例如：用于获取第一个元素的First()方法
用于访问下一个元素的next()方法，用于判断是否有下一个元素的hasNext()方法，用于获取当前元素的currentItem()方法等，在具体的迭代器中实现这些方法。

2.ConcreteIterator(具体的迭代器)：它实现了抽象的迭代器接口，完成对聚合对象的遍历，同时在具体迭代器中通过游标
来记录在聚合对象中所处的当前位置，具体实现时，游标通常是一个表示位置的非负整数。

3.Aggregate(抽象聚合类)：它用于存储和管理元素对象，声明一个createIterator()方法用于创建一个迭代器对象，充当抽象迭代器工厂角色。

4.ConcreteAggregate(具体聚合类)：它实现了在抽象聚合类中声明的createIterator()方法，该方法返回一个与该具体聚合类对应的
具体迭代器ConcreteIterator实例。
